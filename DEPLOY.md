# 短信工具

一款简洁高效的短信编辑和发送工具，完全不存储任何用户数据，保护您的隐私！

## 🔒 隐私保护

- ✅ **不存储用户数据**：所有数据仅在浏览器内存中，关闭页面即消失
- ✅ **不收集个人信息**：无需注册、无需登录
- ✅ **不追踪用户行为**：无任何统计和追踪
- ✅ **本地处理**：短信内容仅在本地编辑，不上传到任何服务器

## 🚀 功能特点

- 📝 快速编辑短信内容
- 📤 一键调用手机短信应用
- 🔗 生成分享链接，方便他人使用
- 📋 预设常用短信模板

## 🚀 国内推荐部署方式

### 方案一：使用 Gitee Pages（推荐，国内访问快）

1. **注册 Gitee 账号**
   - 访问 https://gitee.com/
   - 免费注册（国内平台，访问速度快）

2. **创建仓库并上传代码**
   - 创建一个新的 Gitee 仓库
   - 将项目代码上传到仓库

3. **开启 Gitee Pages**
   - 进入仓库的「服务」->「Gitee Pages」
   - 选择部署分支（通常是 main）
   - 点击「启动」即可
   - 免费提供 https://您的用户名.gitee.io/仓库名 这样的链接

### 方案二：使用 Vercel 中国版（同样简单）

1. **注册 Vercel 中国版账号**
   - 访问 https://vercel.digital/ 或搜索「Vercel 中国」
   - 使用 GitHub/Gitee 账号注册

2. **一键部署**
   - 导入您的仓库
   - 点击 Deploy，等待部署完成

### 方案三：使用 GitHub Pages + jsDelivr CDN（国内加速）

1. 将代码推送到 GitHub 仓库
2. 在仓库设置中开启 GitHub Pages
3. 使用 jsDelivr CDN 加速访问：
   - 原始地址：https://您的用户名.github.io/仓库名
   - 加速地址：https://cdn.jsdelivr.net/gh/您的用户名/仓库名@main/

### 方案四：使用腾讯云/阿里云对象存储（稳定可靠）

1. 注册腾讯云/阿里云账号（新用户有免费额度）
2. 创建对象存储桶
3. 上传构建后的文件
4. 开启静态网站托管
5. 配置 CDN 加速（可选）

## 📦 构建步骤

无论选择哪种方案，都需要先构建项目：

```bash
npm run build
```

构建后的文件在 `dist` 文件夹中，将该文件夹的内容上传即可。

## 📱 使用分享功能

部署完成后，您就可以：

1. 在网站上编辑好短信内容（可选填收件人号码）
2. 点击「分享链接」按钮
3. 复制生成的链接
4. 通过微信/短信/任何方式分享给家人朋友
5. 他们打开链接后，内容已自动填好，只需点击「发送短信」即可！

## 💡 示例场景

**帮老人编辑短信：**
- 您帮父母编辑好给亲戚的节日祝福短信
- 生成分享链接，通过微信发给父母
- 父母在手机上打开链接
- 点击「发送短信」，自动打开手机短信应用
- 选择收件人发送即可

**团队协作：**
- 编辑好通知短信
- 分享链接给同事
- 同事一键发送给自己的客户

## 🔥 临时快速分享方案（无需部署）

如果您只是想临时测试或给少数人使用，可以使用以下方法：

### 方案：使用内网穿透工具（推荐用 cpolar/ngrok）

1. **安装 cpolar**（国内访问快）
   - 访问 https://www.cpolar.com/
   - 下载安装免费版本

2. **启动内网穿透**
   ```bash
   # 在项目目录运行
   npm run dev -- --host 0.0.0.0
   
   # 新开一个终端，运行
   cpolar http 5173
   ```

3. **获取公网链接**
   - cpolar 会给出一个公网访问链接
   - 这个链接可以分享给任何人访问

**注意**：免费版链接会定期变化，适合临时使用，长期使用建议部署到静态托管。

## 🎯 最简单方案推荐

如果您想快速上手，我推荐按以下顺序尝试：

1. **最快上手**：先试试 cpolar 内网穿透，立即就能分享
2. **长期使用**：注册 Gitee 账号，用 Gitee Pages 部署（稳定、免费、国内访问快）
